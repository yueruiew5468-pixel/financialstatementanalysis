# 財報分析助手

這是一個財報指標分析工作台，重點是把公開財報資料轉成可追溯的比率、異常偵測、風險分數與分析摘要，而不是讓 LLM 直接憑空撰寫報告。

## 已完成內容

- 企業客戶切換與雙鴻科技公開財報資料
- 獨立財務分析核心 `analysis-engine.js`
- 關鍵財務比率與週轉指標計算
- 異常變動偵測規則與追蹤建議
- 資料口徑切換：年度、季累計、單季、TTM
- 風險分數：流動性、槓桿、現金流、獲利、營運資金五構面
- 正式財報分析 memo 結構
- AI 財報分析摘要草稿資料生成
- 來源追溯與防幻覺提示
- 人工註記欄位
- 客戶檔案直接載入公開資訊資料
- CSV / JSON 結構化財報測試匯入
- CSV 範本下載
- 分析摘要複製與報告匯出

## 如何使用

直接打開 `index.html` 即可使用，不需要安裝套件或啟動服務。預設公司為雙鴻科技股份有限公司（3324），公司檔案會直接載入公開財報資料進行分析；CSV / JSON 可作為結構化測試資料匯入。

## 資料來源策略

本系統採用「公司檔案資料直接分析」原則：

- 上市櫃公司可由公司檔案直接載入公開財報資料。
- 系統依資料來源整理財務數據、計算比率、偵測異常並生成分析摘要。
- 所有結論需保留來源、欄位與人工註記。
- 公開資訊資料在此流程中是直接資料來源，不是比對資料。

## 資料欄位

CSV / JSON 測試匯入使用下列結構化欄位：

```text
year,revenue,grossProfit,operatingIncome,netIncome,currentAssets,inventory,accountsReceivable,currentLiabilities,totalLiabilities,totalAssets,equity,interestExpense,operatingCashFlow
```

## 建議正式版架構


## 目前前端分析核心

`analysis-engine.js` 負責所有可驗證的計算與規則：

- `enrichStatements`：排序年度資料並計算流動性、槓桿、獲利、現金流、應收帳款與存貨指標。
- `detectAlerts`：依固定門檻與同比變動偵測營收下滑、負債增加、現金流轉負、應收帳款壓力、存貨水位與償息能力等異常。
- `evaluateRecommendation`：將高、中風險警示轉成初步分析結論與信心分數。
- `buildReviewSections`：把計算結果整理成分析摘要段落，前端只負責呈現。
- `buildTtmRows`：以最近四季單季資料計算 TTM，資產負債表項目保留最新期末數。
- `calculateRiskProfile`：產生 0-100 風險分數與五構面拆解。
- `detectQuarterlyAlerts`：偵測季度現金流、短期借款、DSO、存貨天數與營運資金異常。

## MOPS 下載資料

3324 雙鴻科技的公開資訊觀測站原始下載與整理檔放在 `data/mops/3324_auras/`。目前包含 2023Q1-2025Q4，以及 2026Q1 的合併資產負債表、合併綜合損益表與合併現金流量表。

統整原則：

- 資產負債表使用各季期末數。
- 綜合損益表第 2、3 季同時有當季與累計欄；年度檔使用第 4 季年度累計。
- 現金流量表使用年初至當季累計；單季現金流以累計差分計算。
- 前端雙鴻樣本資料已改用 `data/mops/3324_auras/parsed/auras_annual_app_ready_million_twd.csv` 的年度統整數字。

## 分析資料口徑

目前前端採用「年度 + 季度 + TTM」混合口徑：

- 年度資料：作為三年趨勢與基本面分析基礎。
- 單季資料：用於異常偵測，抓現金流轉負、存貨與應收帳款快速增加等早期警訊。
- 季累計資料：保留 MOPS 原始累計口徑；前端趨勢圖與比率矩陣採「同季累計」比較，例如最新為 Q1 時，比較 2023Q1、2024Q1、2025Q1、2026Q1，避免把 Q4 年累計直接接到隔年 Q1。
- TTM 近四季：用於主要比率矩陣與分析結論，降低單季季節性造成的誤判。

年度、季累計、單季、TTM 的切換會同步影響 KPI、財務趨勢圖與財務比率矩陣；其中季累計不是連續季度趨勢，而是同季累計趨勢。

雙鴻頁面會載入 `data/mops/3324_auras/auras_mops_data.js`，因此直接開 `index.html` 也能使用 MOPS 季度資料，不需要啟動服務或 fetch CSV。

可用 `tools/build_auras_mops_static.py` 從 `mops_3324_full_table_amounts.csv` 重建前端靜態資料與統整 CSV，不必重新連線下載。

## 驗算測試

```text
node tests/analysis-engine.test.js
```

測試涵蓋：

- Q2 / Q3 損益表使用 MOPS 當季欄，不誤用累計欄。
- Q4 單季 = 全年 - Q1 - Q2 - Q3。
- 單季現金流 = YTD 差分。
- TTM = 最近四季單季數加總。
- 資產負債表項目使用期末數，不做跨季加總。
- 短期借款欄位可供財務風險規則使用。

## 防幻覺設計原則

- 財務比率由程式計算，不交給 LLM 心算。
- LLM 產出必須引用原始財報來源、Excel 儲存格或資料來源。
- 報告區分「已驗證數據」、「系統推論」與「需人工判讀」。
- 資料不足時輸出「無法判斷」，不得自行補齊。
- 高風險結論不可自動決策，必須保留人工判讀。
